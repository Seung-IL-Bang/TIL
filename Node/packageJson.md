# package.json 이란?

npm 명령어를 통해 개발에 필요한 라이브러리 의존성 및 프로젝트에 필요한 정보들을 기록한 파일이다. 개발 과정에서 절대 지우면 안된다.

<br>

1. name : 프로젝트 이름

2. version : 현재 프로젝트 버전

3. description : 프로젝트 설명

4. main : 현재 프로젝트를 패키지화 하여 npm 생태계에 업로드할 때 필요한 옵션이다. 다른 프로젝트에서 import 할 때 해당 main 속성의 자바스크립트 파일을 가져와서 사용하게 된다.

5. **scripts : 현재 프로젝트 내부에서 사용할 수 있는 스크립트 명령어들을 명시할 수 있다.**

6. devDependencies or dependencies
: 개발자가 직접 npm install을 통해 설치한 패키지들의 의존성 기록 및 관리한다. devDependencies는 개발할 때만 필요한 라이브러리 의존성만 기록된다.

그 외 프로젝트와 관련된 키워드, 저자, 라이선스 등 명시 가능하다.
